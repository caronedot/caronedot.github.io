本地搭建 challenge

从 github 拉取 ctf-2024 项目，按照 README 文件使用 docker 启动 wombocombo，启动前需要把 images 里的 challenge 删掉，避免启动到别的项目

漏洞分析

该题使用的是 EIP712 标准，签名后通过 Forwarder 合约转发给 Staking 合约

由于改写了 _msg_sender 函数，正常情况是在进行 onlyOwner 解析时会摘取 calldata 最后的地址做校验，但是 Staking 继承了 Multicall ，在这个函数将 Forward 构造的 calldata + from 当作一个正常的 calldata 解析，所以我们把 owner 塞到 calldata 去进行调用就能够绕过 onlyOnwer 的校验

解题思路

发放的函数 notifyRewardAmount(uint256 _amount) 是需要 owner 才能调用的，我们可以利用 Multicall 进行 owner 身份的伪造进行调用，然后等待 20s 后进行 getReward() 获取 rewardsToken

对于 EIP712 的签名，需要构造 structHash + domainSeparator，需要注意的是 structHash 里面的参数是不能变的，不然计算出来的签名会改变，像下面的写法计算出来的签名肯定是不一样的
