本文章为 openzeppelin-ctf-2024 系列，记录 alienspaceship 题目的思路与解答

用到的工具

  • docker
  • foundry
  • make
  • vscode

本地搭建 challenge

从 github 拉取 ctf-2024 项目，按照 README 文件使用 docker 启动，启动前需要把 images 里的 challenge 删掉，避免启动到别的项目

题目解析

该题目采用字节码进行合约生成，需要分析合约原有代码和调用 missionAborted 方法，官方推荐了一个字节码转合约的网站：dedaub，为了方便直接看官方的源合约

漏洞分析

该题目不存在明显漏洞，都是一些比较正常的调用

解题思路

如果要过这一关，调用 missionAborted 方法需要满足以下条件

  1. 距离 < 1000km
  2. 当前载重 < 1000kg
  3. 访问过 51 区

距离 < 1000km

调用 jumpThroughWormhole 缩短距离，前提是取得 CAPTAIN 身份，载重 > 1000kg

当前载重 < 1000kg

调用 dumpPayload 减少载重，前提是取得 ENGINEER 身份

访问过 51 区

调用 visitArea51 进行访问，前提是取得 CAPTAIN 身份

结论

综上，我们得先取得 CAPTAIN 身份，再创造一个合约取得 ENGINEER 身份就能通过这关
